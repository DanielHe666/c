# 项目开发及学习经历
## 在线c语言编译器
项目地址： https://chenyuheee.github.io/c
本地路径： /users/hechenyu/projects/c

项目描述：
零配置、开箱即用的在线 C 语言编译/运行环境，完全托管于 GitHub Pages。前端单页整合 Monaco Editor、代码运行、测试用例管理、度量分析、命令面板等功能，结合 Service Worker 提供基础离线体验。运行全部委托给 Wandbox API，失败时在界面内返回真实错误信息。支持在浏览器本地持久化代码、主题与测试用例，并可生成包含输入/用例的分享链接。

技术栈：
- 前端：纯原生 HTML/CSS/JavaScript + Monaco Editor（通过 CDN loader 按需加载）
- 状态持久化：localStorage 保存代码、主题、测试用例
- 运行服务：Wandbox 编译 API（HTTPS POST，超时 8~10s）
- 分享机制：Base64 编码 JSON 写入 URL hash（包含代码 / stdin / 测试用例）
- PWA：手写 Service Worker 缓存核心静态文件
- 版本管理与部署：Git、GitHub Pages（主干直发，无构建流程）

实现方式：
1. 通过 Monaco 官方 AMD loader 在浏览器端加载编辑器，动态切换暗/明主题、撤销/重做、命令面板
2. 纯前端实现 C 语法智能缩进与格式化（栈 + 状态机），并在“自动缩进”按钮中调用
3. 利用 localStorage 同步代码、主题、测试面板用例，保持跨刷新持久化；关闭标签页前自动保存
4. 运行按钮向 Wandbox 发送编译请求，展示 stdout/stderr/exit code，并在失败时反馈真实错误
5. 分享窗口将代码、stdin、测试用例序列化为 JSON 后 Base64 编码写入 URL hash，实现零后端分享
6. 测试用例面板基于 Wandbox 编译结果批量校验输入输出，提供通过/失败汇总
7. Service Worker 预缓存首页、Manifest、图标等静态文件，实现离线打开与资源回源策略

学习技能：
- 前端工程：Monaco Editor 二次封装、命令面板、响应式布局
- 算法实现：针对 C 语法块的缩进/格式化状态机设计
- API 集成：远程编译接口的超时控制、错误提示、多语言提示文案
- 用户体验：快捷键绑定（Ctrl+Enter 运行、Ctrl+S 另存为）、移动端侧栏切换、I18n 切换
- PWA 与缓存：手动编写 Service Worker、版本号控制和缓存失效策略
- 数据处理：JSON 序列化分享、Base64 编码/解码、localStorage 容错

收获和推广：
- 通过 GitHub Pages 零成本上线，解决社团同学随时练习 C 语言的需求
- 掌握了 Monaco Editor 深度定制经验，可快速移植到其他语言沙箱
- 建立版本化脚本（`scripts/version.js`）统一展示版本号，形成稳定的迭代节奏
- 引入测试用例面板、代码度量、命令面板等增强功能，提升教学场景体验
- 当前线上版本 v1.3.20，浏览器访问量稳定，持续收集反馈迭代

不足：
- 运行完全依赖 Wandbox，出现超时或限流时只能提示错误，缺少备用编译服务
- Share 链接采用 Base64，超长代码在微信等平台仍可能被截断
- 当前仅支持单文件 C 程序，尚未提供多文件/头文件管理
- 没有多人协作或实时同步功能，暂时定位于个人练习

## AI医学辅助诊断系统
注：本项目和一位巴德年医学试验班的同学合作
项目地址： http://121.43.107.58
本地路径： /users/hechenyu/projects/MedAgt

项目描述：
MEDAGT 智能医学站点——"采集症状，生成可追溯的初步建议"。基于 DeepSeek LLM 与医学数据库检索管线，提供结构化的医学问诊辅助建议。系统支持患者基本信息录入（病例编号、年龄段、性别）、多症状描述（起病时间、严重程度）、补充说明，调用后端推理接口生成初步建议并展示 triage 级别与引用证据。知识库实时同步 PubMed 与 NICE 最新数据，可用于相似病例比对。在返回建议前通过人工校准列表确保结果可追溯和安全。

实现方式：
1. 阿里云服务器部署（IP: 121.43.107.58）
2. 前端：症状录入表单，支持动态添加多个症状项
3. 后端：DeepSeek LLM 作为推理引擎，结合医学数据库检索
4. 知识库：集成 PubMed（英文医学文献）和 NICE（英国临床诊疗指导）
5. 检索：先用Python爬虫下关键字的论文，然后去除带（Vet/Animal）等兽医论文，再根据关键词复现对论文进行排序，再选取top 5提交给LLM
6. 安全机制：有Checker Agent专门对回答进行可靠性检查
7. 结构化输出：triage 分级（紧急程度）+ 引用证据链 以JSON形式回答
8. 多病例管理：支持新建和切换不同病例

技术栈：
- 前端：HTML/CSS/JavaScript，响应式表单设计
- 后端：Python
- LLM：DeepSeek
- 数据库：向量数据库（用于医学文献相似度检索）
- 知识源：PubMed API、NICE 数据接口
- 服务器：阿里云 ECS
- 安全：人工审核规则引擎

学习技能：
- AI Agent应用开发：大语言模型集成、提示工程、医学领域适配
- 算法+LLM减少tokens：检索增强生成、向量数据库、相似度计算
- 医学信息学：医学文献检索、症状编码、诊断分级（triage）
- 数据集成：多源数据（PubMed + CNKI）同步与处理
- 安全与合规：医疗 AI 安全机制、可追溯性设计、人工校准
- 用户交互设计：复杂表单设计、动态字段添加
- API 设计：前后端分离、RESTful 接口

收获和推广：
- 成功构建了具有实际应用价值的医学辅助诊断系统
- 掌握了 RAG（检索增强生成）架构的实际应用
